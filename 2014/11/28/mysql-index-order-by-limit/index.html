where…order by…limit结构是我们最常用的结构之一，例如，获取最近注册的10个用户，积分最高的20个用户列表等，往往也是最容易引起性能问题的。

记住一条：做此类查询时请务必使用索引，否则全表扫描加文件排序将会严重拖慢整个查询

原则

order by…limit…需要综合考虑数据的重复度以及limit的数量using-index-for-order-by-vs-restricting-number-of-rows
